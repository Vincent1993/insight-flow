# InsightFlow POC V2.0 调整版（M1/M2 聚焦）

> 本文档用于将当前工程实现与新目标对齐：优先投入 **M1 单模块智能解读与指纹缓存**、**M2 多模块联动叙事与生成式 Chat**。

## 1. 目标重排

### M1（必须完成）

1. 任意模块可注册、可被侧边栏发现
2. 边缘统计提取（均值 / 斜率 / 离群点）
3. `MD5 Fingerprint` 判定数据变更
4. IndexedDB 缓存命中（未变更数据秒级返回）
5. 数据变更后提示“重新解读”

### M2（必须完成）

1. 多模块勾选聚合上下文
2. Dify 联合分析工作流（变量列表输入）
3. 提示词分层覆盖（内置模板 + 模块预设 + 用户覆盖）
4. 会话回写（sessionId + reply）

### M3（后续）

- 飞书协同闭环、回写链路、AppLink 溯源等暂不阻塞 POC。

## 2. 当前代码映射

- `packages/fingerprint-cache`：M1 指纹与缓存能力
- `packages/compute-engine`：边缘统计计算
- `packages/core`：模块发现、状态总线、回写能力
- `packages/llm-connectors/dify`：M2 会话主路径
- `apps/host-demo`：端到端演示（发现 -> 会话 -> 回写 -> 更新）

## 3. 关键交互策略

### 3.1 指纹缓存判定

1. 用户点击 AI 解读
2. 计算当前 payload 指纹
3. 查询缓存状态：
   - `hit`：直接展示历史结论
   - `stale`：提示“数据已更新，请重新解读”
   - `miss`：调用 Dify 生成并写缓存

### 3.2 提示词分层

最终提示词 = `内置模板层 + 模块预设层 + 用户覆盖层 + 当前问题`

- 内置模板层：`indicator / trend / table / generic`
- 模块预设层：每个模块自定义任务与输出格式
- 用户覆盖层：append / replace

## 4. POC 验收门槛

- [ ] 模块发现链路稳定（注册/选中/同步）
- [ ] 缓存命中率可观测（hit/miss/stale）
- [ ] 同一指纹重复提问不触发二次 AI 请求
- [ ] 数据更新后可触发“重新解读”
- [ ] 多模块联合提问可得到结构化回复
- [ ] sessionId 与 AI 回复可回写页面模块
